# Editor Appearance

This guide covers what controls the look of the writing surface and what you can change yourself.

## What you can change today

**Heading styles** are the main thing you can customize in the app. Use the **Heading Settings** modal (gear menu) to set the size, font, weight, letter spacing, and color of each heading level, per project. See [Headings](headings.md) for details.

## Body text

Body text — the normal paragraphs you write — uses these defaults:

- **Typeface:** IBM Plex Serif
- **Font size:** 15px
- **Line height:** 1.8 (generous spacing between lines, a deliberate choice to keep long-form writing comfortable; the editor never goes below 1.8)
- **Paragraph spacing:** a small gap between paragraphs

Body text appearance isn't exposed as an in-app setting yet — these values are fixed (though a project type template can change them when a project is created). Text color follows your light/dark mode automatically.

> Developer reference (every config key, default, type, and which values are config- vs CSS-controlled): [docs/features/editor/settings.md](../../features/editor/settings.md).
