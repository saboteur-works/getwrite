# Project Types

A **project type** is a template that defines how a new project starts out — which folders are created and which starter documents they contain. Choosing a project type when you create a project gives you a consistent, ready-to-write structure instead of an empty folder.

## Using a project type

When you create a new project from the Start Page, you pick a project type. GetWrite then scaffolds the project's folders and any placeholder documents the type defines, so you can start writing right away.

## Bundled project types

GetWrite ships with these built-in types:

| Type | Use case |
| --- | --- |
| **Blank** | A minimal project with a single Workspace folder — a starting point for custom workflows. |
| **Novel** | Long-form fiction. Includes Workspace, Story Elements, Outline, and Notes folders. |
| **Serial** | Serialized fiction for episodes or installments. |
| **Article** | Short-form articles and essays, with Workspace, References, Sources, and Ideas folders. |
| **Game Writing** | Game writing for video games or tabletop RPGs — Workspace, Research, and Notes folders for lore, characters, and quests. |
| **Poetry and Lyrics** | Poetry and song lyrics, with Workspace, Inspiration, and Experiments folders. |

## Managing project types

Open the **Project Type Manager** from the settings menu (gear icon) to create your own types, edit existing ones, configure their default folders and starter documents, and delete types you no longer need.

> Authoring a project type by hand or integrating with the scaffolding code? See the developer reference at [docs/features/project-types.md](../features/project-types.md) for the JSON format, validation, and runtime behavior.
